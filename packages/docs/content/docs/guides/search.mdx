`search` is the Pentaract's unified search across all entity types. It combines full-text matching with semantic understanding, so queries like "cozy farming games for kids" work alongside exact title lookups like "Elden Ring".

Every request needs an `X-Client-Key` header (see [Quickstart: Register an API client](/docs/getting-started/quickstart#register-an-api-client)).

## Basic search

Pass a query string to `q`. Results come back as a union of summary views — games, profiles, platforms, collections, and engines — ranked by relevance:

<Tabs groupId="lang" items={['TypeScript', 'JavaScript', 'Go', 'Rust', 'cURL']} persist>
<Tab value="TypeScript">
```ts
const url = new URL(
  '/xrpc/games.gamesgamesgamesgames.search',
  'https://gamesgamesgamesgames.games'
)
url.searchParams.append('q', 'grand theft auto')
url.searchParams.append('limit', '10')

const response = await fetch(url, {
  headers: { "X-Client-Key": "API_CLIENT_KEY" }
});
const { results } = (await response.json()) as {
  results: Array<{ $type: string; name: string }>;
};

for (const result of results) {
  console.log(result.$type, result.name);
}
```

## Filtering games

When searching for games, several additional filters are available. All of these only apply to results where the type is `game`.

Multiple values within a filter are combined with OR (a game matching any selected genre will be included). Filters across different parameters are combined with AND (a game must match at least one genre AND at least one mode).

| [GRAC](http://www.grac.or.kr) | South Korea | `grac` | `All`, `Twelve`, `Fifteen`, `Eighteen` |
| [GCRB](https://grcb.or.kr) | South Korea | `gcrb` | `All`, `Twelve`, `Fifteen`, `Eighteen` |
| [ClassInd](https://www.gov.br/mj/pt-br/assuntos/seus-direitos/classificacao-1) | Brazil | `classInd` | `L`, `10`, `12`, `14`, `16`, `18` |
| [ACB](https://www.classification.gov.au) | Australia | `acb` | `G`, `PG`, `M`, `MA15`, `R18`, `X18` |

## Natural language age filtering

The search query itself is analyzed for age-related language. If you don't pass explicit `ageRatings` filters, we'll infer appropriate age limits from the query text:

- `"platformers for kids"` — filters to ratings appropriate for age 7
- `"14yo rpg recommendations"` — filters to ratings appropriate for age 14
- `"family friendly co-op"` — filters to ratings appropriate for age 10
- `"games for teens"` — filters to ratings appropriate for age 13
- `"games for adults"` — filters to ratings appropriate for age 18

This is a convenience for natural-language queries. For precise control, use the `ageRatings` parameter directly.

## Pagination

Search uses cursor-based pagination. The response includes `totalResults` and an optional `cursor`:

<Tabs groupId="lang" items={['TypeScript', 'JavaScript', 'Go', 'Rust', 'cURL']} persist>
<Tab value="TypeScript">
```ts
const url = new URL(
  '/xrpc/games.gamesgamesgamesgames.search',
  'https://gamesgamesgamesgames.games'
)
url.searchParams.append('q', 'rpg')
url.searchParams.append('types', 'game')
url.searchParams.append('limit', '20')

const response = await fetch(url, {
  headers: { "X-Client-Key": "API_CLIENT_KEY" }
});
const page1 = (await response.json()) as {
  results: Array<{ name: string }>;
  totalResults: number;
  cursor?: string;
};

console.log(`${page1.totalResults} total results`);

if (page1.cursor) {
  const url2 = new URL(
    '/xrpc/games.gamesgamesgamesgames.search',
    'https://gamesgamesgamesgames.games'
  )
  url2.searchParams.append('q', 'rpg')
  url2.searchParams.append('types', 'game')
  url2.searchParams.append('limit', '20')
  url2.searchParams.append('cursor', page1.cursor)

  const response2 = await fetch(url2, {
    headers: { "X-Client-Key": "API_CLIENT_KEY" }
  });
  const page2 = await response2.json();
}
```
